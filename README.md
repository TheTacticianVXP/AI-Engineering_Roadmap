# AI/ML Engineering Roadmap — From Scratch to Production

A complete, vetted curriculum for becoming an AI/ML engineer,
starting from Python basics through to VLMs, distributed training,
and production deployment.

## Who This Is For

- Career switchers entering AI/ML with basic Python knowledge
- CS graduates with ML theory but no implementation experience
- Self-learners who want a structured path with vetted resources

## What's Inside

- **Curriculum** — Phase-by-phase study plan with 18 vetted books/courses
  (every resource has Goodreads/Amazon ratings and honest review notes)
- **Two Weekly Plans** — one for studying alongside a job, one for
  full-time self-study
- **Resource Registry** — complete lookup table with ratings, status,
  and consolidated reading plans
- **Learning Prompt** — a system prompt for AI assistants (Claude,
  ChatGPT) that prevents AI dependency and builds real coding ability
- **Chat Reference** — 11 chat templates for different learning contexts
- **Cursor Rules** — IDE rules that enforce code quality and prevent
  AI from writing code you should write yourself

## How to Use

1. Start with `curriculum.md` — read the phase structure
2. Pick your plan: `weekly-plan-office.md` or `weekly-plan-home-study.md`
3. Check `resource-registry.md` for any resource before starting it
4. Use `learning-prompt.md` as your system prompt in AI conversations
5. Use `chat-reference.md` to know which chat to open for what
6. Copy `.cursor/rules/` into your projects

## Key Design Decisions

- **One primary resource per topic.** No paralysis of choice.
- **Every resource vetted.** One book was removed mid-plan after
  discovering 2.0/5 reviews with broken code. See the registry.
- **Active reading estimates.** Times include typing examples and
  writing practice scripts, not just reading speed.
- **Independence Ladder.** The AI prompt system progresses from
  guided learning to peer collaboration over ~12 weeks.
- **Projects solve real problems.** Not toy projects. Each has
  evaluation metrics, testing, and documentation standards.
